## Q31 (Q1.31) — 32-bit signed fixed-point

Q31 (also written Q1.31) is a fixed-point numeric format that represents signed real numbers using 32 bits with 31 fractional bits and 1 integer/sign bit. It's commonly used in DSP and embedded systems where 32-bit integer arithmetic is available but floating-point is costly.

### Bit layout and scaling
- Total bits: 32 (two’s-complement signed integer).
- Format: 1 sign/integer bit + 31 fractional bits.
- Numerical interpretation: treat the 32-bit two’s-complement integer N as the real value N / 2^31.

### Range and resolution
- Minimum representable value: −1.0 (binary 1000...0 = 0x80000000 = −2^31 -> −2^31/2^31 = −1.0).
- Maximum representable value: 1 − 2^(−31) = (2^31 − 1)/2^31 ≈ 0.9999999995343387.
- Resolution (LSB): 2^(−31) ≈ 4.6566128731e−10.

### Representation examples
- 0x40000000 (decimal 2^30 = 1073741824) → 1073741824 / 2^31 = 0.5.
- 0x80000000 → −1.0.
- 0x7FFFFFFF → (2^31 − 1) / 2^31 ≈ 0.9999999995343387.

### Arithmetic rules and common operations
- Addition/subtraction: use 32-bit two’s-complement arithmetic. Overflow wraps; detect or saturate if needed.
- Multiplication:
  - Product of two Q31 values yields a Q62 value conceptually (scaled by 2^62). Practically compute with 64-bit signed integer: int64_t prod = (int64_t)a * (int64_t)b; then shift right by 31 to return to Q31: result = (prod + rounding) >> 31. Handle sign properly and consider overflow/saturation.
  - Typical sequence with rounding:
    ```c
    int32_t q31_mul(int32_t a, int32_t b) {
        int64_t prod = (int64_t)a * (int64_t)b;      // Q62
        int64_t rounded = prod + (1LL << 30);        // add 0.5 LSB for rounding
        int32_t res = (int32_t)(rounded >> 31);      // back to Q31
        // optional saturation logic here
        return res;
    }
    ```
- Shifts:
  - Left shift multiplies by powers of two (watch overflow).
  - Right arithmetic shift divides by powers of two, preserving sign.
- Conversions:
  - Integer to Q31: int32_t q = int_val << 31.
  - Q31 to integer (floor): int_val = q >> 31 (arithmetic shift).
  - Float to Q31: q = (int32_t)roundf(f * 2147483648.0f) but clamp to [−2^31, 2^31−1]; note many implementations use 2147483648.0 (2^31) for scaling and then saturate since +1.0 cannot be represented.
  - Q31 to float: f = q / 2147483648.0f.
- Saturation: when needed (e.g., audio), clamp results to −2^31..2^31−1 to avoid wrap-around.

### Practical notes and use cases
- Use when you need higher precision (31 fractional bits) than Q15 while still using integer arithmetic.
- Common in 32-bit DSPs and signal-processing libraries, audio, filters, fixed-point FFTs.
- Advantages: very fine resolution (~4.66e−10) and wide dynamic precision inside ±1 range. Limitations: limited dynamic range (values restricted to (−1, +1)), careful scaling required to avoid intermediate overflow when accumulating or multiplying many values.

### Example conversions
- Convert 0.25 to Q31: round(0.25 * 2^31) = round(0.25 * 2147483648) = 536870912 → 0x20000000.
- Convert Q31 0x20000000 back: 536870912 / 2147483648 ≈ 0.25.
